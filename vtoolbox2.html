Download¶

The Engineering Vibration Toolbox for MATLAB¶

The Engineering Vibration Toolbox can be installed by downloading vtbud and executing it from within MATLAB. Executing it later also installs or removes the toolbox. You will get an error if vtbud.m is not in your path. If that happens, open vtbud.m in the MATLAB editor and press the F5 key. This will download vtoolbox.zip, expand it in the desired location, and add the path to it to your default MATLAB path. Alternatively, copy and paste this into your MATLAB command console:

» urlwrite('http://www.cs.wright.edu/people/faculty/jslater/vtoolbox/vtbud.m','vtbud.m');vtbud

and press RETURN.

The current revision status of the Engineering Vibration Toolbox can be viewed in the vtbud.html log file.

Troubleshooting¶

A list of updates since 1/1/98 can be obtained for your current version by typing help vtbud in MATLAB. That can be compared to the most up-to-date release.

They will load/run on all platforms supported by MATLAB. The binary files are all .mat files, and the different extensions represent different parts of a project group. To load them type:

» load filename -mat

where » is the MATLAB prompt. [1] The filename must include the extension unless it is .mat.

If you can’t execute any of the .m files (try for example typing “vtbud” from the MATLAB prompt), they are either not in the MATLAB path, or did not get transferred in the correct format (unlikely these days). Use the cd command (like DOS) from within MATLAB to move to the vtoolbox directory. If you can see the files using dir, and still cannot execute them, then try transferring the text files again, being sure that they are transferred in ASCII (text) format.

Removing the Engineering Vibration Toolbox:¶

Type vtbud from the MATLAB prompt.

The Engineering Vibration Toolbox for Octave¶

Download the zip archive for the Octave Engineering Vibration Toolbox. Please read the Readme.rst file inside the archive or refer to it on the Octave Engineering Vibration Toolbox distribution site.

Installation on a Windows machine¶

Unfortunately I haven’t had success recently. This should work, but doesn’t, and I haven’t the time to track the problem down. If you get it all working, please provide me feedback on how to get it to work. In the mean time, my recommendation is to install VirtualBox and install Ubuntu (Linux), then follow the Linux Installation instructions above.

  1. Install Cygwin to start. Now don’t go to fast. Part of the way through you will have the option of installing some packages.

    1. Look in the math section. Select the main Octave distribution and anything else (all is perhaps easiest).

    2. Also select and install xinit, xlaunch, nedit and gnuplot. I am taking this from wiki.octave.org (and other sources), so if it doesn’t work, please look there.

    3. Open the Cygwin terminal.

    4. Type xlaunch.exe and hit return. Answer the questions, open a single window, start a program, don’t use Native OpenGL, launch the application nedit in place of the default xterm.

    5. A window should come up. Open the file .bash_profile and append the following:

      PATH=/usr/bin:/usr/local/bin:/usr/X11R6/bin:/opt/gnome/bin:/opt/kde2/bin
MANPATH=/usr/man:/usr/local/man:/usr/X11R6/man:/opt/gnome/man:/opt/kde2/share/man
INFOPATH='/usr/info;/usr/local/info;/opt/gnome/info'
KDEDIR=/opt/kde2
export PATH MANPATH INFOPATH KDEDIR

    6. Use nedit again to create a file .xinitrc in your home directory in Cygwin. The contents should be:

      exec /opt/gnome/bin/gnome-session

    7. Close Cygwin, relaunch it, and type startx. This should start a desktop environment from which you shoul be able to run Octave.

  2. Put the unzipped ovtoolbox folder in C:\Program Files\GNU Octave #\usr\share\octave\site\m where # is the version number of your Octave installation.

  3. Replace the file C:\Program Files\GNU Octave\#\usr\share\octave\site\m\startup with the file startup in the ovtoolbox directory. This will make Octave behave more Matlab like and help assure greater compatibility. [2]

  4. Type help vtb1 after running Octave to see if the installation of the Engineering Vibration Toolbox worked.

  5. Create a directory C:\Program Files\GNU Octave #\usr\share\octave\site\m\myfiles where you can put your script and program files.

Installation on a Unix/Linux machine¶

Before the Engineering Vibration Toolbox for Octave can be installed, Octave must be installed. Please consult the Octave website for installation instructions. Alternatively, most Linux distributions have it as an easy installation via their package managers. Most pupular, if you are installing Linux for the first time, is probably Ubuntu.

The toolbox can go in any directory as long as the Octave path is set properly in the .octaverc file in the user’s home directory. Like any of the other toolboxes, you should not save personal files inside the ovtoolbox directory since you may inadvertently lose them when you update (i.e. you may decide to delete the directory and replace everything).

Example: On my Mac (which is BSD unix), my .octaverc file contains the command: path(path,"/User/jslater/MyMath/ovtoolbox");

Printing plots¶

To print a plot, the easiest way may be to type print from the Octave console. This should bring up dialog boxes that enable you to send the current plot to your current Windows printer.

The print command can also be used to generate graphics files for embedding into your favorite word processing program. Type help print for details. By default the file will end up in the directory C:\\Program Files\\GNU Octave #\\octave\_files on Windows.

Using the Engineering Vibration Toolbox¶

Typing help vtb?, where ? is a number, will provide a table of contents for the files related to chapter ‘?’. Typing help codename will provide help on the particular code. Note that the filename is codename.m.

Engineering Vibration Toolbox commands can be run by typing them with the necessary arguments just as any other Octave/MATLAB commands/functions. For instance, vtb1\_1 can be run by typing vtb1\_1(1,.1,1,1,0,10). Many functions have multiple forms of input. The help for each function shows this flexibility.
